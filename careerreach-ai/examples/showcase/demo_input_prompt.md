# Demo 输入 Prompt

这是一份可复用的 CareerReach AI Opportunity 工作流输入。它把岗位搜索、硬性筛选、RAG 匹配、结果门槛、Excel 字段和沟通话术规则集中写在一个自然语言任务中。

```text
$boss-job-agent

请使用 boss-job-agent 的 opportunity 工作流，帮我生成一份 AI 产品经理机会表。

目标：
1. 在 BOSS 直聘搜索「AI产品经理」相关正式岗位，城市限定为上海和深圳。
2. 目标数量：20 个合格候选岗位。
3. 这些岗位本身可以是正式岗，但需要判断公司是否可能接受我以 AI 产品经理实习生身份先参与。

硬性筛选：
1. 排除实习岗：岗位名包含“实习/Intern/校招实习”等，或薪资为 100-500 元/天这类日薪。
2. 排除 1000 人以上公司；20-499 人优先；500-999 人可保留但降权。
3. 排除猎头/匿名公司/“某大型公司/某互联网公司”等无法判断真实主体的岗位。
4. 排除已关闭岗位。
5. 排除岗位实际工作地点不在上海/深圳的情况，即使搜索结果城市显示上海/深圳。
6. 如果 JD 写明某能力或工具是“必须项/硬性要求”，而我的 RAG 记忆库中没有相关证据，则一票否决或进入待确认，不要作为合格候选。
7. 对机器人/运动控制/ROS/机械臂、金融/投资/资金/对账、审美/视觉设计等我缺少背景的方向明显降权。

匹配分析：
1. 调用本地 ChromaDB/RAG 记忆库进行匹配，结合我的简历、项目经历、CareerReach Agent 搭建经历和历史补充记忆进行判断。
2. 重点匹配以下方向：AI客服/ToB 场景、Dify 工作流、Agent 搭建、英语能力。
3. 简历匹配度低于 75 或实习接收可能性低于 48 的，不进入最终候选表。

输出 Excel：
表格字段必须包括：公司名、公司主要业务、公司规模、岗位名称、岗位需求判断、工作地点、薪资、一周实习几天、实习时长、简历匹配度、实习接收可能性、推荐等级、状态、匹配理由、风险/待确认、生成的打招呼话术、security_id、job_id、岗位网址。

话术要求：
1. 每个候选公司生成一段独立打招呼话术。
2. 话术要说明由我自己搭建的 CareerReach Agent 基于 JD 和我的 RAG 记忆库生成。
3. 保持 bullet points、简洁、专业。
4. 如果匹配度低于 80，不显示具体分数，只说“匹配度较高”。
```

## 公开演示说明

- 对应输出见 [浏览器预览](demo_output_preview.md) 和 [Excel 工作簿](careerreach_agent_output_public.xlsx)。
- 公开结果使用合成公司、合成岗位标识和 `example.com` 链接，不是实时岗位推荐。
- 真实运行仍需用户本人登录并在本地环境完成；任何联系、回复或投递动作均由用户人工执行。
