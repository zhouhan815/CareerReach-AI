## 与 Prompt 的对应关系

| Prompt 约束 | 公开结果 |
| --- | --- |
| 20 个合格候选 | 20 条，上海 10 条、深圳 10 条 |
| 正式岗位 | 岗位名称不含“实习 / Intern / 校招实习”，薪资均为月薪 |
| 公司规模 | 20-499 人优先；仅保留 3 条 500-999 人候选并明确降权 |
| 匹配门槛 | 简历匹配度均不低于 75，实习接收可能性均不低于 48 |
| 重点能力 | 覆盖 AI 客服 / ToB、Dify、Agent、ChromaDB RAG 与英语能力 |
| 低于 80 分的话术 | 不展示具体分数，统一写为“匹配度较高” |
| 人工审核 | 所有沟通内容都是候选草稿，不自动联系、回复或投递 |

4. 岗位名称
5. 岗位需求判断
6. 工作地点
7. 薪资

9. 实习时长
10. 简历匹配度
11. 实习接收可能性
12. 推荐等级
13. 状态
14. 匹配理由
15. 风险/待确认
16. 生成的打招呼话术
17. `security_id`
18. `job_id`
19. 岗位网址

## 话术样例

> • 您好，这段沟通由我自己搭建的 CareerReach Agent 基于贵司 JD 与我的 RAG 记忆库生成。  
> • 简历匹配度：92/100；Agent 工作流与 AI 客服闭环经验。  
> • 想请问该正式岗位是否可以先以 AI 产品经理实习生身份参与，每周可到岗 4-5 天。

匹配度低于 80 的候选不会在话术中出现具体分数，例如：

> • 匹配度较高；Agent 原型与需求拆解经验。  
> • 想请问该正式岗位是否可以先以 AI 产品经理实习生身份参与，每周可到岗 4-5 天。
